# Logo Direction Brief (Sprint 1)

**Status: brief only — no image asset produced.** This session has no
image-generation tool, so this document hands a human designer (or an
image-capable tool) concrete direction rather than fabricating a placeholder
logo that would just need to be redone. The current abstract "D" mark stays
in place until a real replacement is designed and validated.

## The problem

Villagers reported they can't tell what the current logo (an abstract "D"
shape) represents. An abstract mark forces the viewer to *decode* it; a
concrete mark is *recognized* instantly. For an audience that reads Tamil
first, an English-letter abstraction is a double translation — first "what
letter is that," then "what does that letter mean."

## Direction A — Tamil name as the mark

Make the club's Tamil name (or its initials) *be* the logo, set in a bold,
custom-lettered wordmark — the way many well-known Tamil institutions and
brands do. Villagers read words, not symbols; a wordmark they can literally
read beats a shape they must interpret. Pair it with the green/gold brand
colors already established in the app.

**Reference feel:** a confident, rounded Tamil display letterform, not a
generic system font — similar in spirit to how regional cooperative/club
emblems typically render their name prominently and the icon secondarily.

## Direction B — Literal community symbol

A concrete, recognizable icon instead of an abstract shape:
- **Joined hands / a circle of people** — direct community/friendship symbol.
- **A tree** — ties to Green FYC, recognizable, positive, and works well as a
  small app icon at any size.
- **A simple house/home silhouette** — "digital home of the community," per
  the product philosophy.

Any of these, rendered simply (2–3 colors max, bold outlines, no fine detail
that disappears at app-icon size), beats an abstract geometric mark for this
audience.

## Constraints for whichever direction is chosen

- Must be legible as a **48×48px app icon** on a cheap phone's home screen —
  test at that size before finalizing, not just at presentation size.
- Must work in a single flat color (for use on WhatsApp broadcasts, printed
  materials, letterheads) as well as in the full app palette.
- Keep the existing green/gold identity unless there's a strong reason to
  change it — the logo mark is the problem, not necessarily the color story.

## Validation (do this before finalizing, not after)

Show 3–5 candidate marks to 5 real villagers (a mix of ages) and ask, with no
prompting: **"What is this?"** A mark that most people can name or describe
correctly (even loosely — "it's people," "it's a tree") passes. A mark that
needs explanation fails, no matter how attractive it looks in Figma.

## Next step

Hand this brief to a designer or an image-generation-capable session to
produce 3–5 candidate marks per the directions above, then run the
villager-validation step before picking a final logo.
